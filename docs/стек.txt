Рассмотрим сервера приложений с открытым исхожным кодом, поддерживающие Java EE 7:
	- Glassfish от Oracle
	- WildFly (бывший JBoss) от RedHat


Oracle отказались продолжать коммерческую поддержку сервера GlassFish.
Так как не будет коммерческой версии Oracle, то поддержка и качество GlassFish 4+ по идеи снизится в отличии от
WildFly 8.0 который представляет собой развиваемый сообществом проект, на основе которого создаётся 
сертифицированный для Java EE коммерческий продукт JBoss Enterprise Application Platform.
Коммерческая версия JBoss в отличии от WildFly предоставляет следующие преимущества:
	устойчивость к веб-атакам
	системы кэширования и кластеризации
	JBoss Development Studio – среда разработки новых приложений, основанная на Eclipse
	Пятилетний жизненный цикл поддержки

WildFly имеет большее сообщество чем у Glassfish. Еще одним большим преимуществом WildFly является то,
что для него намного чаще выходят обновления и исправления.




Рассмотрим кроссплатформенные реляционные базы данных:
	PostgreSQL
	MySQL
	MariaDB

PostgreSQL уступает MySQL и MariaDB в скорости работы. Кроме того, в PostgreSQL отсутствует кеш запросов, 
который ускоряет скорость обработки повторяющихся запросов.
В MySQL и MariaDB можно использовать разные движки для таблиц.
MariaDB относительно MySQL имеет улучшеный оптимизатор запросов, улучшеный механизм репликации

PostgreSQL максимально соответствует стандартам sql, имеет лучший механизм транзакий, имеет гораздо больше типов данных, 
позволяет использовать хранимые процедуры.

MySQL и MariaDB лучше работают на высоконадежных системах, включенных 24 часа в сутки 7 дней в неделю. 
В большинстве случаев никаких 'чисток' в MySQL и MariaDB производить не требуется. 
PostgreSQL же пока что не может работать в таких системах, 
так как иногда приходится запускать VACUUM для освобождения занятого последствиями 
работы команд UPDATE и DELETE пространства и проводить статистический анализ, 
необходимый для достижения максимальной производительности PostgreSQL.

Исходя из всего выше сказаного будем использовать MariaDB





Java framework
	Struts 1/2
		Из плюсов можно отметить
			Помощь в обработки и валидации форм
			Относительно простая архитектура

		Недостатки
			Плохая документация
			Морально устарел

	JavaServer Faces (JSF)
		Первый и единственный феймворк от офицального производителя, часть JavaEE
		Большое сообщество, много книг, много курсов
		Большое количество стандартных UI компонентов
		Html и javascript генерируется фреймворком

		Минусы
			Отрисовка произвольного web UI из стандартных компонент может быть очень трудоемка
			Проблемы с совместимостью версий
			Смешивание JSF & JSP
			Сложная настройка security

	Spring framework
		Плюсы
			Быстрый
			Позволяет легко создавать веб-сервисы
			Хорошая документация
			Большое сообщество
			Гибкость: вместо использования монолитного подхода, Spring содержит несколько модулей, которые независимы друг от друга.

		Минусы
			Нету стандартных UI компонентов
			В отличии от JSF весь javascript код приходиться писать в ручную
			Приходится тратить больше времени на верстку

Будем использовать Spring Framework, так так он состоит из компонентов, которые не зависят друг от друга.
Этот подход позволяет выбрать только нужные компоненты и не подключать к нашему приложению лишние библиотеки, 
которые не будут использоваться.


CSS Framework
	Bootstrap
		Плюсы
			Много готовых компонентов
			Большое количество различных дополнений

		Минусы
			Плохая документация для использования препроцессоров

	Foundation
		Плюсы
			Хорошая документация для использования препроцессоров

		Минусы
			Маленькое сообщество
			В сравнении с Bootstrap маленькое количество готовых UI компонентов
	Skeleton
		Плюсы
			Самый легкий

		Минусы
			Маленькое сообщество
			В сравнении с Bootstrap маленькое количество готовых UI компонентов

Все фреймворки имеют хорошую поддержку большого количества браузеров.
Bootstrap и Foundation хорошо подходят для создания адаптивного дизайна.
Так как Bootstrap содержит больше готовых компонентов и существует большое количество дополнений будем использовать его.
